## 🔧 环境配置

- **Python版本**: >= 3.8
- **内存**: 4GB+
- **磁盘**: 2GB+ 可用空间

### Python版本检查
```bash

python --version
```

**如果Python版本 < 3.8**:
- 可以使用轻量版（功能受限）
- 或升级到Python 3.8+以获得完整功能

📖 **详细说明**: 请查看 [PYTHON_VERSION.md](PYTHON_VERSION.md) 了解更多关于Python版本和功能的信息。

## 📦 安装步骤

### 1. 克隆或下载项目
```bash
# 假设已经下载到本地目录
cd /path/to/stock-ai-system
```

### 2. 创建虚拟环境（推荐）
```bash
# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
# Linux/Mac:
source venv/bin/activate
# Windows:
venv\Scripts\activate
```

### 3. 安装依赖

#### 完整版（Python 3.8+）
```bash
# 升级pip
pip install --upgrade pip

# 安装完整依赖
pip install -r requirements_full.txt
```

#### 轻量版（Python 3.6+）
```bash
# 升级pip
pip install --upgrade pip

# 安装轻量版依赖
pip install -r requirements.txt
```

⚠️ **注意**: 轻量版不支持akshare和TensorFlow，功能受限。

### 依赖安装常见问题

#### 问题1: Python版本过低
**解决方案**:
```bash
# 检查Python版本
python --version

# 如果版本 < 3.8，升级Python
# Ubuntu/Debian:
sudo apt install python3.8

# macOS (Homebrew):
brew install python@3.8

# Windows:
# 从 https://www.python.org/downloads/ 下载安装
```

#### 问题2: akshare安装失败（Python 3.6）
**解决方案**:
akshare要求Python >= 3.8，如果使用Python 3.6，只能使用轻量版：
```bash
# 安装轻量版依赖（不含akshare）
pip install -r requirements.txt

# 使用测试版应用
python app_test.py
```

#### 问题3: TensorFlow安装失败
**解决方案**:
```bash
# 如果pip安装失败，尝试使用conda
conda install tensorflow

# 或者降级版本
pip install tensorflow==2.4.0
```

#### 问题4: pandas/numpy版本冲突
**解决方案**:
```bash
# 重新安装pandas和numpy
pip uninstall pandas numpy
pip install pandas==1.1.5 numpy==1.19.5
```

## 🌐 Web应用使用

### 1. 股票查询

#### 功能说明
- 查询A股历史数据
- 查看K线图和统计信息
- 支持自定义时间范围

#### 使用步骤
1. 访问 `/stock` 页面
2. 在股票代码下拉框中选择股票（或输入代码）
3. 设置查询天数（默认30天）
4. 点击"查询数据"按钮
5. 查看结果：
   - K线图（蜡烛图）
   - 统计信息（最高价、最低价、平均价等）
   - 原始数据表

#### 推荐股票代码
- **000001**: 平安银行
- **000002**: 万科A
- **000725**: 京东方A
- **000858**: 五粮液

---

### 2. 模型预测

#### 功能说明
- 使用LSTM深度学习模型预测股价
- 计算预测准确率
- 支持多指标特征分析

#### 使用步骤
1. 访问 `/model` 页面
2. 选择股票代码
3. 设置训练参数：
   - **训练天数**: 建议使用60-180天（太少数据不足，太多训练慢）
   - **预测天数**: 默认5天
   - **测试比例**: 默认20%
4. 点击"训练并预测"按钮
5. 查看结果：
   - 预测曲线（实际vs预测）
   - 预测准确率
   - 未来5天预测价格

#### 训练参数说明

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 训练天数 | 60-180 | 数据太少会导致预测不准确，数据太多会增加训练时间 |
| 预测天数 | 5-10 | 预测天数越长，准确度越低 |
| 测试比例 | 0.2 | 20%数据用于测试，80%用于训练 |

#### 数据不足处理
如果出现"数据不足"错误：
- **原因**: 计算技术指标后，有效数据不足
- **解决方法**:
  1. 增加训练天数（如从60天改为120天）
  2. 更换其他股票
  3. 点击"使用更多数据重试"按钮（系统会自动获取更多数据）

---

### 3. 回测系统

#### 功能说明
- 模拟历史交易
- 计算策略盈利率
- 分析最大回撤和交易胜率

#### 使用步骤
1. 访问 `/backtest` 页面
2. 选择股票代码
3. 设置回测参数：
   - **回测天数**: 建议使用30-90天
   - **初始资金**: 默认100,000元
   - **手续费率**: 默认0.03%
4. 点击"运行回测"按钮
5. 查看结果：
   - 累计收益率曲线
   - 总收益率
   - 最大回撤
   - 交易胜率
   - 交易记录明细

#### 回测策略说明
系统使用以下策略：
1. 当预测价格上涨时买入
2. 当预测价格下跌时卖出
3. 信号确认后次日执行交易

## ❓ 常见问题

### Q1: 启动时提示端口被占用
**原因**: 5000端口已被其他程序使用

**解决方案**:
```bash
# 查找占用端口的进程
# Linux/Mac:
lsof -i :5000
# Windows:
netstat -ano | findstr :5000

# 结束进程或更换端口（修改app.py中的port）
```

### Q2: 数据加载失败
**原因**: 腾讯财经API连接失败或数据源不可用

**解决方案**:
1. 检查网络连接
2. 等待几分钟后重试
3. 系统会自动切换到模拟数据源

### Q3: 模型训练非常慢
**原因**: 训练数据过多或硬件性能不足

**解决方案**:
1. 减少训练天数
2. 使用GPU加速（需要安装CUDA）
3. 降低模型复杂度

### Q4: 预测准确率很低
**原因**: 股票市场具有高度随机性

**解决方案**:
1. 增加训练数据
2. 尝试不同的股票
3. 结合多个技术指标
4. 调整模型参数

### Q5: 内存不足
**原因**: 数据量过大

**解决方案**:
1. 减少查询天数
2. 定期清理缓存（删除data/cache/目录）
3. 增加系统内存

## ⚡ 性能优化

### 1. 使用本地缓存
系统自动将API获取的数据缓存到本地：
- 缓存位置: `data/cache/{stock_code}.csv`
- 有效期: 7天
- 优势: 加快数据加载速度

### 2. 减少查询天数
- 首次查询: 使用30天
- 模型训练: 使用60-120天
- 回测: 使用30-90天

### 3. 清理缓存
定期清理旧缓存以释放空间：
```bash
# Linux/Mac
rm -rf data/cache/*.csv

# Windows
del data\cache\*.csv
```

### 4. 使用虚拟环境
- 隔离项目依赖
- 避免版本冲突
- 方便迁移和部署

## 🎯 最佳实践

### 1. 选择合适的股票
- 选择交易活跃的股票（成交量高）
- 避免停牌或长期不动的股票
- 关注主板股票（沪600xxx、深000xxx）

### 2. 设置合理的参数
- **训练天数**: 60-120天（平衡准确度和速度）
- **预测天数**: 5天（短期预测更准确）
- **测试比例**: 20%（保证训练数据充足）

### 3. 验证预测结果
- 不要只看准确率
- 查看预测曲线的趋势是否合理
- 结合其他分析方法（如基本面分析）

### 4. 风险管理
- **不作为投资建议**: 预测结果仅供参考
- **分散投资**: 不要只依赖一只股票
- **设置止损**: 预测失败时及时止损
